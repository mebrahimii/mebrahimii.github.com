What is a computer cluster ?

A computer cluster is a collection of computers (also called as nodes) connected through high speed local are network that work together to simulate a single much more powerful computer system.


Each node in a computer cluster is controlled by its own operating system. All nodes in a computer cluster a controlled by a specialized scheduler/resource allocator that enforces even

distribution of work among the nodes in a computer cluster.


Each node in a computer cluster performs a different version of the same task.


A difference between computer cluster and computer grid is such that the nodes in a computer grid perform different tasks.


An architecture of computer cluster ranges from a simple two-node system connecting two personal computers to a supercomputer with a cluster architecture that consists

of 50,000 - 100,000 nodes (also called as a warehouse computer).

Computer clusters are used to speed up computing through shared nothing (sharding) partitioning of data and parallelization of data processing on the nodes of a cluster.


Computer clusters provide high availability through automatic replacement of failed node with a replica node.


Advantages of computer clusters: faster processing speed, larger storage capacity, better data integrity, greater reliability and wider availability of resources.


A Linux cluster is a collection of connected array computers that can be viewed and managed as a single system.


A sample computer cluster: Specification of Nova cluster at Iowa State University:

54 regular compute nodes (with two 32-Core Intel 8358 processors, 1.6TB of local NVME storage and 512GB of memory each) and

5 GPU nodes with two 24-Core AMD EPYC 7413 processors, eight A100 GPU cards, 960GB of local storage and 512GB of memory each.

What is cluster computing ?

Cluster computing is the process of sharing the computation tasks among multiple computers included in a computer cluster.


Advantages of cluster computing: cost efficiency, processing speed, expandability, high availability of resources.


At the moment cluster computing is an attractive paradigm for processing large scale science, engineering and commercial applications. Cluster computing

allows for parallelization of computations and data processing with relatively cheap hardware. It is probably the most cost-effective solution for parallelization

of data processing.


Cluster computing requires the specialized algorithms like load balancing, resource sharing and resource scheduling for optimization of data processing on a cluster

of computers.


Cluster computing is an attractive alternative to data processing on large parallel supercomputers. It is mainly due to availability of cheap hardware components

and very simple extensibility of the system.


The simplest configuration of nodes for cluster computing consists of a mater node and slave nodes.

What does Big Data mean and how big is Big Data ?


An informal definition of a term "Big Data" says that "Big Data" is so big that it cannot be stored on the persistent storage devices attached to a single

computer system. Of course it is possible to connect any number of external storage devices to a single computer system and to store data on all devices.

Does not mean that such amount of data is NOT Big Data ? Of course not. If a single computer system is not able to process such amount of data in

a reasonable period of time then it a good reason to say that it is Big Data.


Big Data may also mean an infinite amount of data, i.e. a theoretically infinite sequence of data items.

More modern group of sources for BigData include sources like video documents , streams of messages coming from social websites and email servers and information from various

The largest group of sources of Big Data includes machine generated data. it includes the streams of data coming from sensor attached to industrial manufacturing

lines, satellite data obtained from record the signal coming from outer space, video data and images obtained from recording of real world events.

Machine generated data also include logs, audits, performance traces obtained from operational software systems.

Another way to characterize BigData is through so called nVs features. The original characterization of BigData used 3V features that included

Volume: e.g., billions of rows ? millions of columns, Variety: Complexity of data types and structures, Velocity: Speed of new data creation and growth.


Volume feature means that only very large volumes of data can be called as BigData. Variety feature means that Big Data must come in many different varieties

like text data, images, videos, semi-formatted data, etc. Velocity feature means that only the amounts of data growing and/or changing very fast can be

categorized as BigData.


Over time more V-like features have been added. The additional Vs include

Veracity: Ability to represent and process uncertain and imprecise data,

Value: Data is the driving force of the next-generate business and

Viability: Benefits we can potentially have from data analysis.

Traditional data architectures are based on data warehousing technologies. A data warehouse is a database that contains temporal data, i.e. data collected over

a longer period of time, for example orders submitted by the customers in the last twenty years.


Data warehouse is fed with data coming from the operational databases. An operational database keeps a snapshot of the present information. At the end of a given period of time, typically at the end of a day operational data are appended to data warehouse database. Relational database systems like DB/2, Oracle, MySQL etc

are typical used fro implementation of operational databases. It is possible that operational database contains data in Excel spreadsheet file or even in flat files

controlled by an operating system.


A specialized piece of software is used for data analysis.The outcomes of data analysis are visualized in a form of pie charts, block charts, or functional charts.

Data warehousing technology has been developed over many years and at the moment it is a mature technology that has important strengths.


The strength of traditional data architectures include:

- Centralised governance of data repositories,

- Light-fast inquires performed regularly in daily business,

- Optimisation for OLTP (OnLine Transaction processing) and OLAP (On-Line Analytical Processing),

- Security and access control,

- Fault-Tolerance and backup.


Progresses in management of BigData impose new challenges on data warehousing technology. The challenges for traditional data architectures include:

- New types of data such as unstructured data and semi-structured data,

- Increasingly large amounts of data flowing into organisations,

- New computational paradigms use non-traditional NoSQL databases to rapidly mine and analyse very large data sets,

- Increasing cost of storing and analysing the large amounts of data,

- Increasing use of data analytics, which requires significant storage and processing capabilities.

Hardware for Big Data has two scalability dimensions. It means that it can be scaled-up in two different ways.


Assume that a computer used for processing of all our data provides us wth satisfactory performance. Such computer is represented by an icon located

close to a centre of the system of coordinates. Now, assume that we significantly increase the amounts of data processed and we also increased the total number of

applications processed our data. Now, due to more data and more applications a computer does not provide us with a satisfactory performance.


We have two options. The first option is to purchase a more powerful computer that will be able to deal with the increased workload. Such solution solves

the performance problems. However, it has two important disadvantages. First, a new and more powerful computer costs a lot of money. Second, we have a problem

what to do with the old computer ?


The second option is to purchase more computers like the computer we have now, use fast networks to link the systems and to create cluster of computers.

Such solution better scales up. When the current cluster of computer is not sufficient to process a given workload then we can easily add more systems to a cluster

increasing its processing power. There is no need to remove the earlier purchased systems.

Hadoop, in terms of its developers, is a project that develops open-source software for reliable, scalable, distributed computing. Hadoop runs on a cluster

computers and supports parallel processing of data.


Features of Hadoop

- Capability to handle large data sets, e.g. simple scalability and coordination,

- File size range from gigabytes to terabytes,

Can store millions of those files,

- High fault tolerance,

- Supports data replication,

- Supports streaming access to data,

- Supports batch processing,

- Supports interactive, iterative and stream processing,

- Implements a data consistency model of write-once-read-many access model,

- Runs on commodity hardware, not high-performance computers,

- Inexpensive,

- It can be deployed on premises or in the cloud,

Master-slave architecture of Hadoop clusters.


Hadoop cluster consists of a number of computer systems connected through a fast local area network. One of the nodes in a cluster is a Master node that

controls behaviour of the remaining nodes (Slave nodes). A Master node has a Stand-by master node allocated in order to quickly replace failed Master

node.

Hadoop clusters can support up to 10,000 server and receives near-to-linear scalability in computing power.


A typical Hadoop cluster consists of:

- A set of master nodes (servers) where the daemons supporting key Hadoop frame-works run,

- A set of worker nodes that host the storage (HDFS) and computing (YARN) work,

- One or more edge servers, which are used for accessing the Hadoop cluster to launch applications,

- One or more relational databases such as MySQL for storing the metadata repositories,

- Dedicated servers for special frameworks such as Kafka.

There are two other ways how BigData can be implemented on computer cluster: database clusters and Kubernetes.


A database cluster is a collection of databases that is managed by a single instance of a running database server. A very large database in a database cluster is partitioned over a number of smaller databases each located on a separate node of a computer cluster.


Database clustering requires replication and sharding. Database clustering improve performance, availability, and scalability


The classes of database system that allow for database clustering:

NoSQL systems: MongoDB, RavenDB, Cassandra, Amazon Aurora, ...

NewSQL systems: ClustrixDB, NuoDB, CockroachDB, Pivotal GemFire XD, Altibase, MemSQL, VoltDB, ...

Improved OldSQL systems: Oracle RAC, SQL Server (Windows server Failover Cluster), DB2 Cluster, PostgreSQL, MySQL Cluster, ...
